⚠️ Executive Summary — 9 CRITICAL issues need immediate attention

The app's features and UX are solid. But there are 9 RED criticals spanning security and data integrity that should be fixed before any paid marketing. Top 3: (1) WalletScreen lets any user self-credit unlimited coins via browser console. (2) Agora token endpoint + call_invites RLS together allow anonymous strangers to join active call channels and eavesdrop on private conversations. (3) ProfileScreen saveEditProfile overwrites your bio JSON every time you edit name/about — silently wipes cover_url, notification prefs, sound prefs, expert application.

⭐ Top 5 fixes to ship first

  1. Lock down the coin economy — Move all coin grants/debits behind a server-side SECURITY DEFINER RPC tied to a real Stripe/Razorpay webhook. Right now anyone can sb.from('profiles').update({coins:999999}) from devtools. (~1 day)
  2. Authenticate the Agora token + send-call-push endpoints — Both API routes mint tokens / fire pushes with no auth check. Add Supabase JWT verification + validate against the call_invites row. Also revert AGORA_RLS_LOOSEN.sql. (~half day)
  3. Stop wiping the profile bio — In ProfileScreen.js fix saveEditProfile (L431), Save Profile Info (L978), and Expert App submit (L1805) to re-fetch + merge before writing the bio JSON. Currently every edit silently wipes notif_prefs, sound_prefs, cover_url. (~1 hour)
  4. Wire Profile's onViewUser / onSwitchTab props — ProfileScreen body uses props.onViewUser + props.onSwitchTab but only destructures {session, supabase, onOpenWallet}. Tapping a liker name or "View My Moment" throws ReferenceError. (~5 minutes)
  5. Public post / expert pages + invite-friend loop — Without /post/:id and /expert/:id SSR routes, every share link returns the SPA shell and breaks the share-back loop. This is the biggest growth blocker. (~2 days)

Top 5 highest-ROI additions

  1. Video calls (~3-4 days) — Agora already wired, mostly UI. Transforms RingIn from "voice consultation" to "real expert face-time" — competes with Zoom/FaceTime.
  2. Public post pages + share-back loop (~2 days) — /post/:id SSR route on Vercel + working share + invite-friend bonus. Biggest growth blocker right now.
  3. Voice notes in chat (~2 days) — Universal mobile expectation. Missing it makes RingIn feel incomplete next to WhatsApp.
  4. Coin tipping on posts / Moments (~2 days) — Wallet + billing rails exist. Monetizes free creators. Heart-burst viral moment.
  5. Verified badge + admin moderation dashboard (~3 days combined) — Badge UI is fake right now. Real verification + /admin page gives trust posture for payments to scale.

What works well (so you know the audit was thorough)

HomeScreen + Moments

  • AvatarRing component — clean conditional wrapper, gracefully degrades
  • ReportModal — real Supabase insert with localStorage queue fallback
  • Optimistic-with-snapshot-rollback on every mutation (toggleLike, submitComment, deletePost, saveEditPost)
  • Notification batching — collapses same-type same-post within 30-min window
  • Feed diversity cap — breaks up consecutive same-author posts
  • Moments cube transition — single per-frame transform on cube wrapper, GPU-composited
  • MomentComposer pinch-zoom — direct-DOM transform writes via ref, 60fps
  • Android back-button handling — popstate guard via __ringinMomentDragTime

Messages + Calls

  • Realtime subscription hygiene — every sb.channel has a matching removeChannel cleanup
  • iOS Safari user-activation — mic track created before token roundtrip preserves tap activation
  • Token pre-fetch on incoming ring — cuts ~200-500ms accept latency
  • Conv list caching — localStorage with stale detection
  • Race-condition fix for outgoing channel — pre-generates invite UUID client-side, doc'd clearly
  • CallScreen unmount cleanup — wake lock, listener (refed not lambda), timeouts, audioSession restoration
  • setMuted vs setEnabled — peers don't see user-unpublished which would have ended the call
  • FCM token cleanup on invalid-token — blanks bad tokens automatically

Profile + Settings

  • Sound & Haptics screen — toggles, sliders all persist correctly
  • Hide Like Counts — updates instantly across tabs via storage event
  • Close Friends add/remove — server-backed, optimistic, debounced FTS
  • Notification toggles → push types — write to localStorage + bio + notification_prefs table
  • Password Change — verifies current password via re-signIn before update
  • Delete Account flow — confirm + RPC + signOut + 30-day cooling-off copy
  • Download My Data — real JSON export of posts + comments via Blob download
  • Activity Log — real Supabase fetches + realtime channel with cleanup

Security (secure-by-default)

  • Agora App Certificate kept server-side — only App ID exposed (correct)
  • Firebase Admin lazily initialized — service-account never echoed to client
  • Most migrations 0001-0015 correctly enable RLS + gate on auth.uid()
  • purge_deleted_accounts, refresh_trending_tags — SECURITY DEFINER with revoke from public/authenticated
  • Service worker bypasses caching for Supabase/Agora/Firebase/api — protects auth and prevents poisoning
  • capacitor.config.json — cleartext:false, autoUpdate:false (user approves every OTA)
  • vercel.json — X-Content-Type-Options: nosniff + Referrer-Policy global headers
  • Expand-contract migration pattern documented in CLAUDE.md is properly followed (try/catch around new column reads)
