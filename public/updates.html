Tier 1 Foundations

The basics every app needs — reporting, SEO, performance, safety nets.

✓Real Report flow

Tap "Report" on a post, comment, or user → it now actually saves to Supabase. Before, it just showed a fake "Thank you" alert.

If the database is down, reports queue locally and flush automatically when you come back online.

✓Error boundaries

If a tab crashes, you see a friendly "this tab hit a snag — Try again" card instead of a white screen. The error is logged so I can fix it.

✓"Latest First" feed label

Small label at the top of the feed so people understand new posts appear at the top.

✓Block AI crawlers

Added a robots.txt that tells GPTBot, ClaudeBot, etc. not to scrape your users' posts to train AI models.

✓Schema.org tags

Hidden tags on each page that tell Google "this is a social network app called RingIn." Helps Google rank your site higher.

✓Auto-updating sitemap

A live list of every public post URL that Google reads. Rebuilds on every deploy so new posts get indexed automatically.

✓Pretty link previews

When someone shares a RingIn link on WhatsApp/Twitter/iMessage, it now shows a nice preview card with title + image instead of a plain URL.

✓Image compression on upload

Big photos shrink to ~1.5 MB before upload. Faster posts, cheaper Supabase storage, less data for users on mobile.

✓Read receipts (✓✓ ticks)

WhatsApp-style ticks on your sent messages. One tick = sent. Two grey = delivered. Two blue = read. Each chat can toggle them off in the ⋮ menu.

✓Typing indicators

When the other person is typing, you see the three bouncing dots in real time. Disappears after 3 seconds of inactivity.

⏸Persistent in-call banner

A "you're on a call — tap to return" banner that follows you while you browse other tabs. Deferred because it needs touching the call screen and that's risky.

✓Hide-likes toggle

Settings → Privacy → "Hide like counts." When on, posts show "Liked" or "0 Likes" instead of "247 Likes." Instagram-style.

✓Cursor pagination

Scrolling down the feed loads older posts in chunks of 30 — way faster than loading all posts at once.

✓Tap a hashtag to filter

Tap #fitness in any post → feed shrinks to only fitness posts. Tap "× Clear filter" banner at the top to go back to the full feed.

✓Diversity cap in feed

No more than 3 posts in a row from the same person. Stops one chatty user from drowning your whole feed.

✓Notification batching

If 5 people like your post in 1 minute, you get ONE grouped notification ("5 people liked your post") instead of 5 separate pings.

✓Android notification channels

Android settings let users pick different sounds/vibration for Messages vs Likes vs Calls. Cleaner control over what wakes you up.

✓Migration safety doc

Added rules to CLAUDE.md so future database changes don't break the live app. Two-step deploys for every schema change.

Tier 2 New Features

Bigger features — Close Friends, reactions, privacy, search, account deletion.

✓T2.1 — Restrict mode in chat

Long-press chat header → "⋮" → Restrict User. Their messages still arrive but only they see them. They never know they're restricted. (Instagram's Restrict feature.)

✓T2.2 — Emoji reactions on messages

Long-press any chat message → picker pops up with 6 emojis (❤️ 😂 😮 😢 👍 🔥). Tap one → badge appears under the message.

✓T2.3 — Audience selector on posts

When you compose a post, the chip at the top lets you pick Public / Followers / Private. Your choice gets remembered for next time.

✓T2.4 — Last seen + privacy

Your app pings "I'm online" every 30 seconds. Privacy settings let you hide it from everyone, friends only, or show to all.

✓T2.5 — Moment view counts

"👁 23" badge appears on your own moments showing how many people viewed it. Tap → see the full list of viewers (your moment, only you see this).

⏸T2.6 — Polls in moments

Backend is ready. UI deferred — lowest priority. Tell me when you want it.

✓T2.7 — Close Friends + green ring

Settings → Privacy → "Manage Close Friends" → search and add people. When you compose a moment, toggle ★ Close Friends ON → only they see it, and your avatar gets a green ring. Instagram-style.

✓T2.8 — Global search

Search bar at the top of Search tab. Type anything → finds matching people AND posts. Uses Postgres full-text search so it's fast even with millions of rows.

✓T2.9 — Real delete account

Settings → bottom → "Delete Account." Now actually marks your account for deletion with a 30-day cooling-off period (log in again within 30 days to cancel). Hard-deletes after that.

⏸T2.10 — Age gate on signup

Backend is ready. UI deferred — Supabase Auth needs a small customization to enforce age 13+ at signup.

✓T2.11 — Server-side blocks

Blocked users now stored in the database (not just your phone). So if you block Bob on phone A, he's also blocked on phone B. Your old localStorage block list auto-migrates the first time you open the new build.

✓T2.12 — Trending hashtags

Open Search tab with an empty box → strip of trending hashtags ("#fitness", "#startup", etc.) — what's hot in the last 24h. Tap → search filtered to that tag.

✓T2.13 — Notification prefs

Settings → Notifications. Per-type toggles (Likes / Comments / Messages / Calls / Follows). Now writes to database too, so settings sync across devices.

Emergency Bug Fixes High severity

Real bugs caught during testing/audits that were blocking core flows.

1
Search → Message button crashed

When you searched for an expert and tapped Message, it crashed with "props is not defined." Fixed — Message now opens the chat correctly.

2
Search → Call didn't ring the person

Calling someone from search wasn't actually inviting them — your end started a call but theirs never rang. Fixed — call_invites is now properly written so the callee gets the ring.

3
Unblock button unblocked everyone

In Profile → Blocked Users, tapping Unblock on one person was removing ALL blocked users at once due to a type mismatch. Fixed — now only removes the one you tap.

4
Home tab — "selected tag is not defined"

Tapping a hashtag chip caused "home tab hit a snag" because the hashtag filter state was declared in the wrong place. Fixed.

5
Home tab — "hide likes is not defined"

Same scope class as bug #4 — the Hide Likes preference hook was declared inside the user profile component instead of the main feed. Fixed in ringin-v3.1.apk.
